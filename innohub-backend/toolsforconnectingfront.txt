BACKEND TO FRONTEND INTEGRATION REFERENCE

1. PROJECT OVERVIEW

Project type:
- NestJS LMS backend
- Prisma 5 + PostgreSQL
- JWT auth + Firebase login
- Active runtime modules from AppModule:
  - Auth
  - Users
  - Courses
  - Topics
  - Enrollments
  - Progress
  - Admin Statistics

Important runtime facts:
- Global API prefix: /api
- Swagger docs: /docs
- CORS: enabled, origin=true, credentials=true
- Global validation:
  - whitelist=true
  - forbidNonWhitelisted=true
  - transform=true

Meaning for frontend:
- Every API path below is prefixed with /api
- Unknown request fields can cause 400 or be stripped depending on validation metadata
- Query params with @Type(() => Number) are auto-converted from strings

Modules/files that are not part of active frontend integration:
- src/app.controller.ts exists, but AppModule currently has no controllers array entries, so root GET /api is not active
- No active uploads/media/settings/notifications modules found
- No active file upload endpoints found

2. AUTHENTICATION AND AUTHORIZATION

Auth types:
- Bearer JWT for protected endpoints
- Firebase ID token accepted only by POST /api/auth/firebase

Protected header format:
- Authorization: Bearer <accessToken>

JWT behavior:
- Token payload contains:
  - sub
  - email
  - role
- On authenticated requests, request.user is a safe user object, not raw Prisma user
- Password hash is never attached to request.user

Authorization model:
- Public endpoints:
  - GET /api/courses
  - GET /api/courses/:id
  - GET /api/topics
  - GET /api/topics/:id
  - POST /api/auth/register
  - POST /api/auth/login
  - POST /api/auth/firebase
- Auth-only endpoints:
  - GET /api/auth/me
  - POST /api/auth/logout
  - GET /api/users/me
  - PATCH /api/users/me
  - POST /api/enrollments
  - GET /api/enrollments/my-courses
  - GET /api/enrollments/my-courses/:courseId
  - POST /api/progress/lessons/:topicId
  - POST /api/progress/lessons/:topicId/complete
- Admin-only endpoints:
  - GET /api/users
  - GET /api/users/:id
  - POST /api/courses
  - PATCH /api/courses/:id
  - DELETE /api/courses/:id
  - POST /api/topics
  - PATCH /api/topics/:id
  - DELETE /api/topics/:id
  - GET /api/admin/statistics/summary
  - GET /api/admin/statistics/recent-users
  - GET /api/admin/statistics/recent-enrollments

Admin roles:
- ADMIN
- SUPER_ADMIN

Guard behavior:
- JwtAuthGuard requires a valid bearer token
- RolesGuard checks request.user.role against @Roles(...)

3. GLOBAL API CONVENTIONS

Base URL:
- /api

Content type:
- application/json

Validation behavior:
- DTO fields use class-validator
- Query/page/limit number fields are transformed from string to number
- isPreview query in topics is transformed from "true"/"false" string to boolean
- Search query params are trimmed
- Email values are normalized in AuthService using trim().toLowerCase()
- Course slugs are auto-generated if not provided

Pagination format:
- Used by users, courses, topics, enrollments
- Standard response shape:
  {
    "items": [...],
    "meta": {
      "page": number,
      "limit": number,
      "total": number,
      "totalPages": number
    }
  }

Message-only responses:
- POST /api/auth/logout
- DELETE /api/courses/:id
- DELETE /api/topics/:id

Error response format:
- Standard NestJS HttpException shape is expected:
  {
    "statusCode": 400,
    "message": "Validation or business error message",
    "error": "Bad Request"
  }
- Validation errors may return message as string[] depending on the failure

5. DTO REFERENCE

DTO Name: RegisterDto
File: src/auth/dto/register.dto.ts

Fields:
- email: string | required | must be valid email
- password: string | required | min length 6
- fullName: string | required | non-empty
- avatarUrl: string | optional | must be valid URL
- role: 'ADMIN' | present in class but no validation decorators, not used by service

Validation:
- email -> @IsEmail
- password -> @IsString, @MinLength(6)
- fullName -> @IsString, @IsNotEmpty
- avatarUrl -> @IsOptional, @IsString, @IsUrl

Notes:
- AuthService ignores any role coming from client and always creates USER
- Because global validation uses whitelist/forbidNonWhitelisted, this stray role field is a frontend pitfall and should not be sent

Example payload:
{
  "email": "john@example.com",
  "password": "StrongPassword123",
  "fullName": "John Doe",
  "avatarUrl": "https://cdn.example.com/avatar.jpg"
}

7. FRONTEND FIELD MAPPING NOTES

AUTH
- Backend expects fullName, not name
- Backend expects idToken for Firebase login, not token
- Do not send role on register despite the stray property in RegisterDto

USERS
- PATCH /users/me expects fullName, avatarUrl, password
- avatarUrl must be a valid URL if sent
- There is no separate displayName field

COURSES
- Backend expects imageUrl, not image
- Backend expects icon, not thumbnail
- Backend expects durationLabel, not duration
- Backend expects totalLessons as number
- Backend expects createdBy if you want to override creator; otherwise omit it
- Backend returns publishedTopicsCount and enrollmentsCount for frontend cards

TOPICS / LESSONS
- Topic is the backend lesson resource
- Backend expects lessonNumber, not order
- Backend expects videoId, not videoUrl
- Backend expects durationLabel, not duration
- Backend expects contentMarkdown/contentHtml, not description
- Backend uses isPreview boolean

ENROLLMENTS
- To enroll, send courseId only
- my-courses list returns progress nested under progress, not at root

PROGRESS
- Client sends absolute watchSeconds
- Do not send incremental seconds
- POST /progress/lessons/:topicId updates watch progress only
- POST /progress/lessons/:topicId/complete is the explicit completion action

ADMIN
- recent-users and recent-enrollments use raw limit query param string, not a dedicated DTO

ENUM UI MAPPING EXAMPLES
- CourseLevel:
  - UI "Boshlang'ich" -> BEGINNER
  - UI "O'rta" -> INTERMEDIATE
  - UI "Yuqori" -> ADVANCED

8. COMMON VALIDATION ERRORS

Likely causes of 400 Bad Request:
- Sending unknown DTO fields because forbidNonWhitelisted=true
- Sending empty string for fields validated by @IsUrl
- Sending translated enum labels instead of raw enum values
- Sending page/limit as non-numeric strings
- Sending lessonNumber/order as 0 or negative
- Sending watchSeconds as negative
- Sending invalid email format
- Sending password shorter than 6 chars
- Sending isPreview query as something other than true/false
- Sending duplicate (courseId, lessonNumber) when creating/updating topic

Likely causes of 401 Unauthorized:
- Missing Authorization header on protected routes
- Invalid/expired JWT
- Invalid Firebase ID token
- Invalid email/password

Likely causes of 403 Forbidden:
- Using USER role on admin-only endpoint
- BLOCKED or PENDING user trying to authenticate or learn
- Trying to update lesson progress without being enrolled

Likely causes of 404 Not Found:
- Requesting unpublished course/topic via public GET
- Enrolling into non-existent or non-published course
- Accessing my enrolled course when enrollment does not exist
- Progress update against non-existent topic

10. FRONTEND INTEGRATION CHECKLIST

- Use /api as the base prefix for all requests
- Send Authorization: Bearer <token> on protected routes
- Store and reuse accessToken from auth responses
- Use raw enum values from backend, not translated labels
- Send fullName for user registration/profile
- Send idToken for Firebase login
- Send lessonNumber for topics, not order
- Send imageUrl and icon for courses, not image/thumbnail
- Treat /courses and /topics public endpoints as published-only catalogs
- Use paginated responses as items + meta
- Do not assume all admin endpoints have query DTOs; recent admin stats only accept limit
- Expect decimals like progressPercent to come back as JS numbers in mapped responses

QUICK FIXES FOR FRONTEND

1. Register form:
- Send fullName, not name
- Do not send role

2. Firebase login:
- Send idToken, not token

3. Course form:
- Send imageUrl, not image
- Send icon, not thumbnail
- Send durationLabel, not duration

4. Topic form:
- Send lessonNumber, not order
- Send videoId, not videoUrl
- Send contentMarkdown/contentHtml, not description

5. Progress tracking:
- Send absolute watchSeconds
- Do not send deltas/increments

6. Query params:
- page and limit must be numbers
- isPreview must be true/false

7. Public catalog:
- Frontend should not expect draft courses/topics from GET /courses or GET /topics
